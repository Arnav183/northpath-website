services

What we do

Model evaluation and red-teaming
Rubrics that independent raters apply the same way, and red-teaming for unsafe or evadable behavior. The mistakes we catch most often:
  • made-up citations
  • arithmetic errors
  • answers built on the wrong numbers
  • claims the source does not back up
  • answers written to pass the rubric, not to be right
Training data and human feedback
The human feedback post-training runs on:
  • preference rankings
  • rubric-scored ratings
  • written critiques of model responses
  • audits of existing datasets for label noise
AI adoption advisory
Where a model fits a team's workflow, and where it fails quietly. The output is specific:
  • which tasks to hand to a model
  • what to measure before launch
  • what to watch after
Custom AI tooling
Small tools that ship as plain code your team can run and change:
  • eval harnesses
  • annotation interfaces
  • scripts that turn raw outputs into reviewable data

work

Selected engagements

Anonymized summaries of recent engagements. Client names and project specifics are withheld under NDA.

Preference data for reward-model training
Domain experts compared model responses pairwise across mathematics, statistics, and code. Each comparison shipped with:
  • a preference direction and strength
  • a written justification
  • every claim verified, with math recomputed and code rerun
Eval design for document-grounded assistants
Evaluation tasks for assistants that answer questions over realistic document sets: statements, forms, and spreadsheets. Every task carried:
  • a weighted rubric
  • graded reference responses
  • documented failure modes
Stress-test tasks for research agents
Research questions that current models answer confidently and wrong, each needing synthesis across several independent sources. What we delivered:
  • a verified answer
  • the full evidence trail behind it
Review and quality control of annotation work
Reviewers re-checked other annotators' rankings and failure-mode reports against the underlying artifacts before anything was approved. Two things came out of each pass:
  • corrections verified against the source
  • guideline fixes, so the same mistakes stopped recurring
Communication-quality grading for assistant models
Graders scored candidate responses on tone, voice, and formatting against a shared anchor scale, kept consistent by:
  • calibration logs across the team
  • a cited rationale for every score

All of this work runs on tooling we build ourselves: capture and verification utilities, scoring panels, and local dashboards, shipped as plain code. If you have a project like these, send a short note.

about

About

Northpath Consulting is an AI development and consulting firm in the Atlanta, Georgia area. The team does the work itself: reviewing model outputs, writing evals, producing feedback data, and building tooling.

Our client work spans frontier AI labs, early-stage startups, data vendors, and engineering teams putting models into production. Deliverables arrive in the format the pipeline needs: JSONL datasets, rubric documents with anchors and worked examples, and eval reports with a failure taxonomy.

Engagements are scoped and direct: a defined deliverable, a fixed window, and a senior consultant accountable for the result. Most projects run a few weeks inside the client's existing tools and channels.
