One evaluation task, start to finish

This walkthrough shows the shape of a single task from our evaluation pipeline: the prompt, two model responses, the rubric, the judgment, and the record that ships. A real dataset is this, several hundred times over.

fully synthetic example · client tasks are covered by NDA

The task

Every task pairs a prompt with source material and a verifiable ground truth. Here the assistant must compute from a table, which makes every claim in its answer checkable.

prompt

Using the table, how much did we exceed the cloud budget across Q1, and which month was worst?

source document: cloud spend, Q1
monthbudgetactual
January$12,000$13,850
February$12,000$11,600
March$14,000$16,730

Note the trap: February came in under budget, and the question does not say how to treat that. A good response resolves the ambiguity instead of ignoring it.

Two model responses

The same prompt goes to two models. Read both. One of them contains two verifiable failures.

response A

Across Q1 you exceeded the cloud budget by $6,180. January was $1,850 over, February was $1,600 over, and March was $2,730 over. March was the worst month, as confirmed in the April reconciliation row of the table.

response B

Net across Q1, spend was $4,180 over budget: January ran $1,850 over, February came in $400 under, and March ran $2,730 over. Counting only the months that exceeded budget, the overrun is $4,580. March was the worst month. The table covers January through March only.

  • Arithmetic slip. The table shows February at $11,600 against a $12,000 budget, which is $400 under. Response A reports it as $1,600 over and carries the error into its total.
  • Fabricated citation. There is no April reconciliation row. The table ends at March. Confident references to sources that do not exist are one of the most common failures we document.

The rubric

Responses are graded against weighted criteria written before any response is read. Each criterion has pass and fail anchors, so two raters grading independently should land on the same score.

criterionanchorsweight
Arithmetic verified against the table pass: every figure recomputable from the source. fail: any value that does not check out. 3
Cites only rows that exist pass: every reference maps to a table row. fail: any invented source. 3
Resolves the under-budget ambiguity pass: states how February is treated. fail: silently picks one reading. 2
Answers both parts of the question pass: gives a total and names the worst month. 1

The judgment

Each response is graded per criterion, then the pair gets a preference direction, a strength, and a written justification. Nothing ships until the math is recomputed and every citation is checked.

preference: B · strength 3 of 3

A misreads February as an overrun and cites a reconciliation row that does not exist. B recomputes cleanly, states how the under-budget month is treated, and stays inside the source. Both failures in A were verified against the table before this judgment shipped.
